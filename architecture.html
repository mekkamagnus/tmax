Key Design Decisions

Emacs Split-Brain Architecture

Following Emacs, the TypeScript core handles performance-critical work (terminal I/O, buffer management, rendering) while T-Lisp handles all editor behavior (commands, modes, key bindings, extensibility). This means new features can be added via T-Lisp without touching TypeScript.

Immutable Gap Buffer

Text is stored in a functional gap buffer implementation (FunctionalGapBuffer) that returns new instances on mutation rather than modifying in place. This pairs with the React rendering model and enables cheap undo/redo snapshots.

React as Dumb View

The React + Ink layer is purely presentational. All logic flows through the Editor class and T-Lisp. React components read state and call editor methods — they never contain business logic.

Zero Dependencies

The entire editor has zero external npm dependencies (Ink is bundled). The Bun runtime provides the standard library. This keeps startup fast and the bundle minimal.

Operation Module Pattern

Editor API functions are organized as independent factory functions (createBufferOps, createCursorOps, etc.) that receive an EditorStateAccess interface and return a Map<string, TLispFunctionImpl>. The createEditorAPI function composes all modules into a single registry.
